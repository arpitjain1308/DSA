# Peak Index in a Mountain Array

## 🧠 Approach

- Used **binary search** to find the peak element

- Maintain:
  - `start` → starting index (1)  
  - `end` → ending index (n - 2)  

- Traverse while `start <= end`:
  - Find `mid`

- Check if `mid` is peak:
arr[mid-1] < arr[mid] && arr[mid] > arr[mid+1]

- If true → return `mid`


### 🔹 If not peak:

- If array is still **increasing**:
arr[mid-1] < arr[mid]
→ move right (`start = mid + 1`)

- Else:
→ move left (`end = mid - 1`)


## 💡 Key Insight

- Mountain array:
- strictly increasing → peak → strictly decreasing  

- If you are on increasing side → peak is on right  
- If you are on decreasing side → peak is on left  


## 🧩 Pattern

This problem follows **Binary Search on Answer / Peak Finding** pattern.

### 🔹 Used when:
- Array has a peak or turning point  
- Monotonic change (increasing → decreasing)  
- Need `O(log n)` solution  


## ⏱ Complexity

- **Time Complexity:** `O(log n)`  
- **Space Complexity:** `O(1)`


## 💻 Code

```cpp
int peakIndexInMountainArray(vector<int>& arr) {
  int start = 1;
  int end = arr.size() - 2;

  while (start <= end) {
      int mid = start + (end - start) / 2;

      if (arr[mid-1] < arr[mid] && arr[mid] > arr[mid+1]) {
          return mid;
      }
      else if (arr[mid-1] < arr[mid]) {
          start = mid + 1;
      }
      else {
          end = mid - 1;
      }
  }

  return -1;
}
