# 33. Search in Rotated Sorted Array

## 🧠 Approach

- Used **modified binary search** to handle rotated sorted array

- Maintain:
  - `start` → beginning index  
  - `end` → ending index  

- Traverse while `start <= end`:
  - Find `mid`
  - If `nums[mid] == target` → return index

- Key idea:
  - At any point, **at least one side (left or right) is sorted**

---

### 🔹 Case 1: Left part is sorted

nums[start] <= nums[mid]

- Check if target lies in left part:

nums[start] <= target <= nums[mid]
- If yes → search left (`end = mid - 1`)  
- Else → search right (`start = mid + 1`)

---

### 🔹 Case 2: Right part is sorted

- Otherwise, right side is sorted

- Check if target lies in right part:

nums[mid] <= target <= nums[end]
- If yes → search right (`start = mid + 1`)  
- Else → search left (`end = mid - 1`)

---

## 💡 Key Insight

- Even though array is rotated:
- One half is always **sorted**
- Use that sorted half to decide where target can exist

---

## 🧩 Pattern

This problem follows **Modified Binary Search** pattern.

### 🔹 Used when:
- Array is **sorted but rotated**
- Need to search in `O(log n)`  
- Direct binary search doesn’t apply fully  

---

## ⏱ Complexity

- **Time Complexity:** `O(log n)`  
- **Space Complexity:** `O(1)`

---

## 💻 Code

```cpp
int search(vector<int>& nums, int target) {
  int start = 0;
  int end = nums.size() - 1;

  while (start <= end) {
      int mid = start + (end - start) / 2;

      if (nums[mid] == target) return mid;

      if (nums[start] <= nums[mid]) { // left sorted
          if (nums[start] <= target && target <= nums[mid]) {
              end = mid - 1;
          } else {
              start = mid + 1;
          }
      } else { // right sorted
          if (nums[mid] <= target && target <= nums[end]) {
              start = mid + 1;
          } else {
              end = mid - 1;
          }
      }
  }

  return -1;
}
