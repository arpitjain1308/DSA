# 9. Palindrome Number

## 🧠 Approach

- Check basic edge cases:
  - If number is negative → return `false`
  - If number is single digit → return `true`

- Reverse the given number:
  - Store original number in `b`
  - Use variable `a` to build reversed number

- Extract digits using modulo: d = b % 10

- Build reversed number: a = a * 10 + d

- Reduce original number: b /= 10

- After reversal:
- Compare reversed number `a` with original `x`
- If equal → palindrome  
- Else → not palindrome  


## 💡 Key Insight

- A number is palindrome if it reads same forward and backward  
- Reversing the number and comparing is a straightforward way to check  


## 🧩 Pattern

This problem follows **digit manipulation / number reversal** pattern.

### 🔹 Used when:
- Need to process digits of a number
- Problems involving reverse, sum of digits, etc.


## ⏱ Complexity

- **Time Complexity:** `O(log n)`  
- **Space Complexity:** `O(1)`


## 💻 Code

```cpp
bool isPalindrome(int x) {
  if (x < 0) {
      return false;
  }
  else if (x > 0 && x <= 9) {
      return true;
  }

  long a = 0;
  long b = x;

  while (b > 0) {
      int d = b % 10;
      a = a * 10 + d;
      b /= 10;
  }

  if (a == x) {
      return true;
  }

  return false;
}
